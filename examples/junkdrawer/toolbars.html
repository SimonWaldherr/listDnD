Toolbars

This example is broken in Internet Explorer

Go ahead, re-arrange the buttons.

 
 

The horizontal toolbar is built from an identical list as the vertical toolbar. Different CSS and a horizontally contrained drag script make the difference.

Firefox isn't respecting the ":active" pseudo-selector on the list elements. When you mousedown on a button it should look pressed in. This is because we capture "onmousedown" to initiate a drag. Returning true from the onmousedown handler gives the right behavior for ":active", but breaks the dragging script. It's possible to work around this with more JavaScript, I just haven't gotten to it yet.
